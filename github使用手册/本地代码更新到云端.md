# 🚀 Git 操作指南：本地代码关联远程仓库

### **场景说明**：本地代码为最新版本，但尚未与 GitHub 建立联系。

---
```
1️⃣ 初始化本地仓库
在项目根目录 `D:\snf\git\sniffy_app` 下运行，生成 Git 监控配置文件：

git init
2️⃣ 关联远程仓库
告诉本地 Git 目标服务器的地址（代号为 origin）：

git remote add origin [https://github.com/iambillfang-dev/sniffy_app.git](https://github.com/iambillfang-dev/sniffy_app.git)
3️⃣ 提交本地代码存档
将本地所有文件暂存并提交到本地仓库：

git commit -m "Initial sync: local code is newer than cloud"
4️⃣ 强制同步云端历史（核心步骤）
由于本地是新库，需强制拉取云端代码进行“合体”：

git pull origin main --allow-unrelated-histories
💡 冲突处理：如果提示 Conflict，请手动打开文件保留最新代码，然后运行：
如果出现vim编辑器，直接esc--wq--回车
git add .
git commit -m "解决合并冲突，保留本地最新版本"
5️⃣ 推送到 GitHub
如果报错分支问题git push -u origin master

将合并后的最终代码上传至云端：

git push -u origin main

6️⃣ GitHub 网页端后续操作
如果推送后网页未直接更新，请检查以下两点：

A. 处理分支“双胞胎” (Main vs Master)
现象：网页显示 2 branches，代码推到了 master 但首页显示的是空的 main。

解决：

点击网页上的 Settings -> Branches。

点击两支箭头图标（Switch），将 Default branch 修改为你刚才推送的分支（如 master）。

B. 合并 Pull Request
操作：看到黄色横条提示 "Compare & pull request" 时点击它。

确认：确保显示 Able to merge。

执行：点击 Create pull request -> Merge pull request -> Confirm merge。

✅ 完成！现在你的 GitHub 仓库已与本地同步。
