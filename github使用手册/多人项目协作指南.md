# 协作指南
1.项目负责人建立组织或添加协作者
个人项目： 进入仓库 Settings -> Collaborators -> Add people，输入队友的 GitHub ID。队友接受邀请后，就有了直接推送（Push）代码的权限。

组织项目： 建议创建一个 Organization，将队友加入团队中，这样管理权限更专业。

2. 克隆仓库到本地
检查你的 GitHub 注册邮箱，会有一封来自 GitHub 的邀请邮件。

点击 "View Invitation" 并接受，打开终端执行：

Bash
git clone https://github.com/用户名/项目名.git
3. 分支管理（协作的核心）
千万不要所有人都在 main 分支上乱改！ 正确做法是每人负责一个功能，就开一个新分支：

创建并切换分支： git checkout -b branch-r

# 修改代码后提交：
1. git add .
2. git commit -m "完成了登录功能"

推送到 GitHub： git push origin branch-r

4. 合并代码（Pull Request）
在 GitHub 网页上，你会看到一个黄色的提示框，点击 Compare & pull request。

这本质上是发起一个“申请”：“我写好了，请求合并到主分支”。

队友可以查看你的代码，发表评论，确认没问题后点击 Merge pull request。

# 更新云端代码到本地（强行将本地更新覆盖云端）
git init
git remote add origin <repository_URL>
git checkout -b branch-wzr
git add .
git commit -m "feat: commit local modifications"
git push -f origin branch-wzr
# 更新云端代码到本地（本地也有更新）
1. git stash      暂存本地修改，执行后，你的工作区会变得干净，本地修改被安全地藏起来了
2. git pull origin main     重新拉取云端代码
3. git stash pop     把刚才暂存的本地修改恢复出来
4. 如果不想要暂存的： git stash drop
   
如果想不要本地的了
1.git restore .撤销所有本地未提交的修改
2.git pull origin main 拉取


补充模式二：开源/跨团队协作（Fork 模式）
适用于你没有目标仓库的写入权限，但想贡献代码的情况。

Fork： 点击别人仓库右上角的 Fork，这会在你的账号下创建一个一模一样的副本。

修改： 在你自己的副本里随便改。

PR： 改完后，点击 New Pull Request。这会将你的代码修改申请发送给原作者。

协作中的“救命”指南：处理冲突
当两个人在同一个文件的同一行做了不同的修改时，Git 会“懵掉”，这就是冲突（Conflict）。

如何解决：
拉取最新代码： git pull origin main

寻找标记： Git 会在代码里插入这样的符号：

Plaintext
<<<<<<< HEAD
你的修改
=======
队友的修改
 main
手动抉择： 删掉这些符号，保留正确的代码，然后重新 add 和 commit 即可。

# 补充代码
1. 简略视角：只看改了哪些文件
如果你只想快速确认一下上一次提交是否包含了你修改的那个脚本：

Bash
git show --stat
输出内容：上一次 commit 的作者、日期、提交信息，以及受影响的文件列表和每个文件增删的行数统计。

2. 详细视角：看具体的代码改动 (Diff)
如果你想逐行检查代码是怎么改的（就像在 GitHub 上看 PR 一样）：

Bash
git show
操作小贴士：如果改动很多，屏幕会进入阅读模式。按 空格 翻页，按 q 退出查看。

3. 历史清单视角：看最近几次提交的记录
如果你想看看之前好几次都提交了什么（确认老板的提交和你自己的提交）：

Bash
git log --oneline -n 5
输出内容：最近 5 次提交的简短哈希值和提交信息。

进阶用法：如果你想看这几次提交分别动了哪些文件，加个参数：git log --oneline --name-only -n 5。
